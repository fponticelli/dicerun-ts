> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Architecture:** Extend existing component patterns (OneOfUnionType, OneOfLiteral) to handle new discriminated union variants. The app uses @tempots/dom with Signals for reactive rendering. No framework changes - just add new branches to existing components.

**Tech Stack:** TypeScript, @tempots/dom, Vite, dicerollerts v0.7.0

- [ ] **Step 2: Create CountThresholdView**

```tsx
export function CountThresholdView ({ range }: { range: Signal<Range> }): JSX.DOMNode {
  return <span>{range.map(r => {
    switch (r.type) {
      case 'exact': return `= ${r.value}`
      case 'value-or-more': return `>= ${r.value}`
      case 'value-or-less': return `<= ${r.value}`
      case 'between': return `${r.minInclusive}..${r.maxInclusive}`
      case 'composite': return r.ranges.map(rangeToString).join(', ')
    }
  })}</span>
}

function rangeToString (r: Range): string {
  switch (r.type) {
    case 'exact': return `= ${r.value}`
    case 'value-or-more': return `>= ${r.value}`
    case 'value-or-less': return `<= ${r.value}`
    case 'between': return `${r.minInclusive}..${r.maxInclusive}`
    case 'composite': return r.ranges.map(rangeToString).join(', ')
  }
}
```

Note: We need `Range` imported as a value type (not just type) for the `rangeToString` function parameter. It's already imported as `type Range`.

### Task 5: Use parseWithErrors for better error messages

**Files:**
- Modify: `src/reducer.ts`
- Modify: `src/state.ts`
- Modify: `src/components/ExpressionInput.tsx`

Currently the reducer uses `DiceParser.parse()` and on failure shows raw `DecodeError[]` from partsing. Switch to `DiceParser.parseWithErrors()` which provides structured `ParseError` objects with messages, positions, and suggestions.

- [ ] **Step 1: Update state.ts**

Change the `ParseError` state type to use the new error format. Replace the `partsing/error` import:

```ts

```

Update the `ParseError` interface:

```ts

  type: 'parse-error'
  source: string
  err: DiceParseError[]
}
```

Update the `Expression.parseError` factory:

```ts

  return { type: 'parse-error', source, err }
}
```

Remove the `import { type DecodeError } from 'partsing/error'` line.

- [ ] **Step 2: Update reducer.ts**

In the `evaluate-expression` case, replace `DiceParser.parse()` with `DiceParser.parseWithErrors()`:

```ts
case 'evaluate-expression':
{
  const result = DiceParser.parseWithErrors(action.expr)
  if (!result.success) {
    return {
      ...state,
      roll: null,
      expression: Expression.parseError(action.expr, result.errors),
      probabilities: null
    }
  } else {
    const validated = DE.validate(result.value)
    if (validated == null) {
      const expression = Expression.parsed(action.expr, DE.toString(result.value), result.value)
      const [rollResult, nextSeed] = roll({ ...state, expression })
      return {
        ...state,
        roll: rollResult,
        seed: nextSeed ?? state.seed,
        expression,
        probabilities: null
      }
    } else {
      return {
        ...state,
        roll: null,
        expression: Expression.parsedInvalid(action.expr, validated, result.value),
        probabilities: null
      }
    }
  }
}
```

Update the import to include `DiceParser`:

```ts
import { DiceParser, DE, Roller, type RollResult } from 'dicerollerts'
```
